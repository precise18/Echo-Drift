# Project Overview

The 10-minute orientation for Echo Hunt 1.0.0: what the game is, how
the code is shaped, and where to read more about any piece. If you're
new to Godot itself, read BEGINNER_GODOT_GUIDE.md first.

## The game in one paragraph

Two players on a LAN. Each round, one hides and one hunts in a
mirror-symmetric arena. The twist: the Hider's movement is continuously
recorded and replayed 10 seconds later as a glowing, audible ghost —
the Hunter tracks the past to find the present, and a clever Hider
feeds their own future ghost lies. First to three round wins takes the
match.

## Architecture in one diagram

```
                    ┌─ AUTLOADS (session state, always alive) ─────────────┐
                    │ NetworkManager   ENet lifecycle, reconnect sessions  │
                    │ RoundManager     one round: roles, clock, win checks │
                    │ MatchStateManager score, phase, first-to-3           │
                    │ MapManager       map registry + host->client sync    │
                    │ AudioManager     buses, music/wind, stings           │
                    │ GameSettings     persisted player prefs              │
                    │ TransitionScreen loading covers                      │
                    └───────────────┬──────────────────────────────────────┘
                                    │ signals / RPCs
      ┌─ SCENES ─────────────────────┴────────────────────────────┐
      │ MainMenu.tscn   router: title/host/join/settings/credits  │
      │ Main.tscn       game root: map container, players,        │
      │                 MultiplayerSpawner, EchoSystem, HUD       │
      │   └ EchoChamber.tscn  the arena (built by its script)     │
      │   └ Player.tscn        CharacterBody3D + camera + sync    │
      │   └ EchoGhost.tscn     the replayed past (no collision)   │
      └──────────────────────────────┬───────────────────────────┘
                                     │ composed from
      ┌─ KITS (class_name statics, pure builders/logic) ──────────┐
      │ MapKit        walls/pillars/lights/particles/navigation   │
      │ SoundFactory  every sound, synthesized + cached           │
      │ UIKit         the theme + every widget                    │
      │ WinConditions / RoleManager / SpawnManager  pure rules    │
      └───────────────────────────────────────────────────────────┘
```

Three tiers: **autoloads** own session state, **scenes** own the tree,
**kits** own reusable construction and pure logic. Almost nothing is an
asset file — geometry, audio, and UI theme are all generated by the
kits (the ten CC0 Kenney props are the exception).

## Design principles that explain most decisions

1. **Server decides, everyone derives.** Only the host ends rounds or
   assigns roles (reliable RPCs); but anything computable from
   replicated state — footsteps, particle bursts, victory jingles, the
   match-over conclusion — is computed locally on every peer with zero
   extra network traffic.
2. **One owner per concern.** Changing a sound touches one function in
   SoundFactory; a color, one constant in UIKit; a rule, one small
   pure-logic class. CONTRIBUTING.md maps concerns to owners.
3. **The theme is load-bearing.** The mirror symmetry of the arena,
   the cyan glow shared by ghost/pool/pads/UI accent, and the
   hollow-vs-dry footstep design all restate the core mechanic: your
   past is present in the world.
4. **Everything verified headless.** Every pass shipped with a
   two-process host/join regression (TEST_PLAN.md) — that pattern
   repeatedly caught real races before players could.

## The echo mechanic, mechanically

`EchoRecorder` samples the Hider's transform + animation 10×/s into a
rolling 10 s buffer — independently on every peer, from already-
replicated positions (recording costs no bandwidth). `EchoGhost` renders
the buffer at `now − 10 s` with interpolation (one binary-search lookup
per frame), plus a synthesized hum, hollow footsteps, and a world-space
particle trail. `EchoSystem` owns one recorder and N ghosts, so
multiple simultaneous echoes replay one shared history. Full detail:
ECHO_SYSTEM.md.

## Production-readiness statement

1.0.0 is considered ready for a game-jam release: full audit clean (no
debug output, no dead files, licensing verified — LICENSES.md),
production builds exported and boot-verified for Linux and Windows,
automated regressions green, and the remaining imperfections are
documented rather than hidden (KNOWN_ISSUES.md). The single remaining
pre-publish gate is human: one real two-machine LAN match on the
exported builds, per TEST_PLAN.md §Cross-machine.
